# Sustainability

Idep is Apache 2.0, free forever. This document explains how the project funds
the people who build it — and what it will never do to get there.

---

## Four non-negotiables

1. **The editor is free, always.** No feature gating, no license keys, no telemetry.
2. **Apache 2.0 is permanent.** No relicense, ever.
3. **Revenue funds contributors, not founders.**
4. **Any paid layer is optional and privacy-preserving.**

---

## How it's funded

**Tier 1 — GitHub Sponsors + Open Collective** *(active from day one)*  
Transparent, public. Companies using Idep in production can sponsor publicly.
Individuals can back the project directly. All income and expenses are visible.

→ [idep on Open Collective](https://opencollective.com/idep-editor) *(coming soon)*  
→ [idep on GitHub Sponsors](https://github.com/sponsors/idep-editor) *(coming soon)*  
→ [Discord community](https://discord.gg/PAKTq7YsMK)

**Tier 2 — OSS ecosystem grants** *(target: 200 GitHub stars)*  
Anthropic, HuggingFace, NLnet, Sovereign Tech Fund, and ASEAN digital economy
programs. Idep directly drives adoption of open AI backends — that's a fundable
argument. Grant applications go out when the star count and contributor activity
justify them.

**Tier 3 — Issue bounties** *(target: working alpha)*  
Specific issues with attached rewards, funded from the Tier 1 pool.
Infrastructure via [Algora](https://algora.io). Bounties are always scoped with
a written specification — no ambiguous prizes.

**Tier 4 — Hosted team RAG** *(target: first org using Idep in production)*  
The one paid product: a shared embedding index for teams, so multiple engineers
share codebase context without per-developer re-indexing. The AI backend is
still configured locally by the user — no routing through Idep servers.
$15–20/seat/month. 100% of revenue enters a contributor pool.

---

## Contributor pool model

When Tier 4 revenue exists, it is distributed monthly to active contributors.
The distribution model is decided by contributors before the first dollar
arrives — not retroactively. Options under consideration:

- Commit-weighted split (automatable, transparent)
- Governance vote among contributors with 10+ merged PRs
- Flat maintainer split + bounty allocation for one-time contributors

The model will be documented here once agreed.

---

## What this project will never do

- Sell or gate editor features behind a subscription
- Introduce AI credits or quota systems
- Collect telemetry to measure usage (the codebase is Apache 2.0 — it would be
  found immediately)
- Take institutional investment with equity
- Change the license
